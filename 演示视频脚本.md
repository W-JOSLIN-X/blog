## 详细脚本

### 一、开场（0:00 - 0:25）🎬

> **画面**：浏览器打开 https://haop11.github.io，展示首页全貌

大家好，这是我的信息技术实践课设作品——一个基于 Jekyll 和 GitHub Pages 的个人博客系统。

整个站点采用玻璃拟态暗色主题设计，所有页面都是完全自主编写，没有使用任何现成模板。下面我来演示一下各个功能模块。

---

### 二、首页概览（0:25 - 1:05）🏠

> **画面**：从上到下滚动首页

首页采用了聚合式布局，把站点的核心内容集中展示。最上面是最新随笔，每篇卡片显示分类标签和摘要，点击可以直接跳转到详情页。

往下是最新照片，六张照片网格排列。注意看，点击任意一张照片，会带上它的分类参数跳转到照片墙，自动筛选出同主题的照片。

再往下是最新项目，展示技术栈标签，点击可以跳转 GitHub。

右侧是个人信息侧边栏，显示头像、简介和四个维度的统计数据——说说、随笔、照片、项目的数量一目了然。

> **画面**：指一下侧边栏的统计数字

---

### 三、随笔模块（1:05 - 1:40）📝

> **画面**：点击随笔卡片，进入详情页 → 返回列表页

点击任意随笔卡片，进入详情页。每篇随笔都是独立的 Markdown 文件，由 Jekyll Collection 机制生成专属 URL。

可以看到标题、发布日期和分类标签。正文支持完整的 Markdown 语法——标题层级、列表、代码块、链接等。

点击"返回随笔列表"回到列表页，这里会展示所有随笔。这个模块的设计思路是把长文内容和短动态分开管理，随笔适合写读书笔记、技术总结这类有一定深度的内容。

---

### 四、说说模块（1:40 - 2:15）💬

> **画面**：通过侧边栏导航进入说说页

说说模块是类似微博的短内容区域。左侧有一条渐变时间线，每条说说是一张全宽玻璃卡片。

每张卡片包含心情图标和标签——支持开心、思考、运动、发现等九种心情分类——还有发布时间和正文内容。

> **画面**：点击点赞按钮

重点演示一下点赞功能。点击爱心按钮，数字加一、按钮变红，同时会爆发出六个爱心粒子特效。关键是——这个点赞数据是持久化的，我现在刷新页面，你看，点赞状态还在。实现原理是用 localStorage 以日期为 Key 存储点赞数据。

---

### 五、照片墙（2:15 - 2:55）📷

> **画面**：进入照片墙页面

照片墙采用自适应网格布局，照片按 1:1 比例展示。顶部分类筛选栏——全部、风景、城市、旅行、自然、生活、动物——点击任意分类，不符合的照片会以缩放淡出动画隐藏，URL 地址栏也会同步更新。

> **画面**：点击"风景"，然后复制 URL 打开新标签

这意味着你可以直接把筛选后的链接分享给别人。比如我把 `photos.html#动物` 发给朋友，他打开后自动显示动物分类的照片。

> **画面**：点击照片打开灯箱

点击任意照片进入灯箱模式，全屏预览大图，底部显示标题描述。可以按 ESC 键或点击遮罩区域关闭。

---

### 六、项目展示 + 关于页面（2:55 - 3:25）🚀

> **画面**：进入项目页 → 关于页

项目页面展示个人作品，每个项目有图标、名称、精选标记、技术标签和描述。点击卡片或下方按钮可以跳转到 GitHub 仓库或在线 Demo。

> **画面**：切换到关于页，滚动展示技能条动画

关于页面包含三个部分。技能区展示 Python、C/C++、Java 的熟练度——注意看，滚动到这一区域时技能条会从零开始动画过渡。经历区是时间线布局，联系方式区点击可以直接复制邮箱和 QQ。

---

### 七、技术亮点（3:25 - 3:55）✨

> **画面**：回到首页，展示背景特效

最后说一下技术亮点。

视觉方面实现了三层背景效果——全屏背景图自动轮换、Canvas 绘制的粒子网络背景、以及十六个独立漂浮动画的萤火虫光点。这些效果全部用原生 CSS 和 JavaScript 实现，没有引入任何动效库。

页面交互上，全局的滚动揭示动画用的是浏览器原生的 IntersectionObserver API，卡片从底部淡入的效果非常流畅。

顶部导航栏采用了毛玻璃效果，滚动时阴影加深，当前页面自动高亮。回到顶部按钮在滚动超过一屏后自动出现。

---

### 八、总结收尾（3:55 - 4:15）🎬

> **画面**：回到首页定格

以上就是我这次实践课设的全部内容。

整个项目从零编写，包含六个页面模块、十几种交互动效，代码托管在 GitHub 上通过 GitHub Pages 自动部署。

感谢观看，如果有任何问题欢迎提问。

## 录制小贴士

- 提前打开浏览器，清理历史记录和缓存，确保页面加载流畅
- 建议使用 1080p 分辨率录制，缩放浏览器到合适大小
- 每个模块切换前停顿 1-2 秒，方便后期剪辑
- 如果某个环节说错，停一下重新说那句话即可，后期可以剪掉
- 可以提前准备好要说的话，但不建议逐字念稿，自然一些效果更好
